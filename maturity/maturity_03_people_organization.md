# マチュリティ定義：人材・組織能力

## 軸の定義

IT部門の人材スキル、組織体制、人材育成の仕組みの成熟度を評価する軸。
「属人化」から「組織能力」への変革、IT/ビジネス両方を理解する橋渡し人材の育成度合いを測定する。

---

## レベル1：属人化型（Ad-hoc）

### 状態
- IT業務が特定の個人（ヒーロー）に依存しており、スキル定義や育成方針が存在しない
- 人材配置は場当たり的で、計画性がない
- IT/ビジネス間の人材交流はほぼ存在しない

### 具体的な特徴

**スキル管理：**
- IT人材の役割やスキル要件が明文化されていない
- 「誰が何をできるか」が属人的な知識に依存
- スキルマップや技術台帳が存在しない

**育成・評価：**
- 体系的な研修プログラムが存在しない
- OJTは場当たり的、メンタリングは自然発生的
- 評価基準が曖昧で、属人的判断に依存

**人材配置：**
- 特定の個人に業務が集中（シングルポイント・オブ・フェイラー）
- 退職・異動時の引き継ぎが困難
- キャリアパスが不明確

**組織文化：**
- IT部門内の知識共有が不足
- ビジネス部門との交流がほぼない
- 「技術屋」として孤立している感覚

**外部依存：**
- 外部ベンダー・派遣に過度に依存
- コア領域の定義なし、すべて外部化の傾向
- ベンダー依存からの脱却意識なし

### チェック項目
- [ ] IT役割のスキル定義書が存在しない
- [ ] 体系的な研修プログラムが存在しない（年間研修計画なし）
- [ ] 過去3年でIT⇔ビジネスのジョブローテーション実績がゼロ
- [ ] IT人材の評価基準が文書化されていない
- [ ] 外部依存度（外部委託費/IT予算）が80%以上

### 典型的な課題
- キーパーソンの退職・異動による業務停滞
- ビジネス要件の理解不足
- IT人材のモチベーション低下
- 採用・定着の困難

---

## レベル2：基礎整備型（Managed）

### 状態
- 主要なIT役割について、スキル要件が定義され、研修・OJT体系が整備され始めている
- ジョブローテーションが試行的に実施されている
- 評価基準が文書化され、運用開始

### 具体的な特徴

**スキル管理：**
- 主要5-6役割（企画/開発/運用/セキュリティ/OT-ITブリッジ等）のスキル定義書が存在
- スキルレベル（初級/中級/上級）の定義
- 推奨資格のリスト化

**育成・評価：**
- 役割別の年間研修計画が策定され、実施記録がある
- OJTプログラムが一部標準化
- IT人材の評価基準が文書化され、運用開始（試行段階含む）

**人材配置：**
- ジョブローテーションが試行的に実施（年1-2名、IT⇔製造現場等）
- 主要業務の担当者が複数名確保され始める
- キャリアパスのドラフトが作成されている

**組織文化：**
- 四半期ごとの知識共有会を開始
- ビジネス部門との定例会議で相互理解が進む
- IT部門内のコミュニケーション改善

**外部依存：**
- コア領域（アーキテクチャ、要件定義等）の定義を開始
- 外部委託の判断基準（競争優位、長期継続、市場スキル）を策定中
- 内製化目標を設定（3年後の外部依存度目標値）

### チェック項目
- [ ] 主要IT役割のスキル定義書が存在する（5-6役割）
- [ ] 各役割の年間研修計画が存在し、実施記録がある
- [ ] ジョブローテーションが過去1年で最低2件実施されている
- [ ] IT人材の評価基準が文書化され、運用されている
- [ ] 外部依存度の方針が明文化されている（コア領域の定義）

### 典型的な課題
- スキル定義の形骸化（実態と乖離）
- 研修の効果測定が不十分
- ジョブローテーション後の知識定着
- 内製化の進捗遅延

---

## レベル3：計画的育成型（Defined）

### 状態
- キャリアパス・ローテーション制度が正式化され、運用されている
- ITリーダーの30%以上がビジネス部門経験を持ち、橋渡し役として機能
- IT人材ポートフォリオが計画的に育成されている

### 具体的な特徴

**スキル管理：**
- 全IT役割のスキル定義が完備され、年次で見直されている
- スキルアセスメントが定期実施（年1-2回）
- 個人別のスキル開発計画が策定されている

**育成・評価：**
- 役割別研修プログラムが充実（社内/外部/e-learning の組み合わせ）
- メンタリング制度が正式化（経験者が若手を支援）
- 評価基準が成熟し、昇進・昇格に明確に連動

**人材配置：**
- ジョブローテーション制度が正式化（期間、対象、評価方法が明確）
- 年間5名以上がローテーション実施
- ローテーション経験者がメンター役を担う
- ナレッジベース（Wiki、事例集等）が整備され、活用されている

**組織文化：**
- IT/ビジネス橋渡し人材が組織内に認知され、活躍している
- 「IT」と「ビジネス」の壁が低くなっている
- チームパフォーマンスの定量モニタリング開始（案件成功率、納期遵守率等）

**外部依存：**
- コア領域は原則内製主導、外部は能力補完・加速のために活用
- 外部依存度が計画的に低減（例：80% → 60%）
- ベンダーとの健全な協力関係（丸投げでなく共創）

### チェック項目
- [ ] キャリアパス・ローテーション制度が正式化され、運用されている
- [ ] ITリーダーの30%以上がビジネス部門経験を持つ（またはその逆）
- [ ] ローテーション経験者のナレッジベースが整備され、活用されている
- [ ] IT人材ポートフォリオ（アーキテクト、BA、OT-ITブリッジ等）が計画的に育成されている
- [ ] チームパフォーマンスの定量モニタリングが開始されている

### 典型的な課題
- ローテーション後の処遇（元の部署に戻る vs 新部署に定着）
- 橋渡し人材の評価・報酬の適正化
- 育成投資と短期成果のバランス
- トップ人材の引き留め

## レベル4：最適化型（Quantitatively Managed）

### 状態
- チームパフォーマンスが定量的にモニタリングされ、改善のPDCAが回っている
- 人材ポートフォリオが戦略的に管理され、将来の事業計画に必要なスキルを先行育成
- ナレッジ共有が文化として定着し、組織学習が機能

### 具体的な特徴

**スキル管理：**
- スキルギャップ分析が定量的（現状 vs 3年後の目標）
- データに基づく人材配置の最適化
- スキルの市場価値との比較

**育成・評価：**
- 育成投資のROI測定（研修費用 vs 生産性向上）
- 個人のパフォーマンスとスキルの相関分析
- 高業績者の特性分析と育成への反映

**人材配置：**
- ローテーションの効果測定（橋渡し機能の定量評価）
- 適材適所の配置が定量データに基づく
- 後継者計画（サクセッションプラン）が確立

**組織文化：**
- ナレッジ共有が評価指標に組み込まれている
- 失敗からの学習が文化として定着
- 心理的安全性が高い組織

**外部依存：**
- 外部との協働が戦略的（コア領域は内製、非コアは外部活用が明確）
- ベンダーとの関係が対等なパートナーシップ
- 外部人材との協働によるスキルトランスファー

### チェック項目
- [ ] チームパフォーマンス指標（案件成功率、品質、納期等）が四半期ごとに測定され、改善されている
- [ ] 人材ポートフォリオが3年後の事業計画に基づき計画的に育成されている
- [ ] ナレッジ共有が評価指標に組み込まれ、活用されている
- [ ] 育成投資のROIが測定されている

### 典型的な課題
- 定量偏重による人間性の軽視リスク
- データ収集・分析の負荷
- プライバシー配慮
- 組織文化の維持・発展

---

## レベル5：継続的革新型（Optimizing）

### 状態
- 将来の事業計画に必要なスキル構成を見据えた戦略的人材マネジメントが行われている
- 変革プロジェクトを自力でリードできる人材層を保持
- IT人材が業界をリードする存在として認知されている

### 具体的な特徴

**スキル管理：**
- 新技術（AI、量子コンピューティング等）への先行投資的育成
- 業界をリードするスキルセットの保有
- 外部（大学、スタートアップ等）との積極的な交流

**育成・評価：**
- 自律的な学習文化が浸透
- 外部発信（カンファレンス登壇、論文発表等）が奨励されている
- グローバルな視点での人材育成

**人材配置：**
- IT/ビジネスの境界が曖昧になり、流動的な配置
- プロジェクトベースの柔軟な組織編成
- 社外との人材交流（出向、兼業等）

**組織文化：**
- イノベーションを推進する文化
- 失敗を許容し、高速で学習サイクルを回す
- ダイバーシティ（多様性）が価値として認識されている

**外部依存：**
- 外部との境界が曖昧（オープンイノベーション）
- ギグワーカー、フリーランスとの協働
- エコシステム型の人材活用

### チェック項目
- [ ] 将来技術への先行投資的育成プログラムがある
- [ ] 外部発信（カンファレンス、論文等）が年10件以上
- [ ] オープンイノベーション・社外協働が年5件以上
- [ ] IT人材が業界内で認知されている（受賞、メディア露出等）

### 典型的な課題
- イノベーションと安定運用のバランス
- コア人材の流出リスク
- 組織のアイデンティティ維持
- 急速な変化への適応疲れ

## レベル間の移行に必要な主要アクション

### 1→2への移行
- 主要役割のスキル定義
- 年間研修計画の策定
- ジョブローテーションの試行

### 2→3への移行
- ローテーション制度の正式化
- ナレッジベースの構築
- IT人材ポートフォリオの計画的育成開始

### 3→4への移行
- チームパフォーマンスの定量モニタリング
- 人材配置の定量的最適化
- 育成投資ROIの測定

### 4→5への移行
- 新技術への先行投資的育成
- 外部発信の奨励
- オープンイノベーション体制構築

## 評価時の留意点

1. **制度と実態の乖離確認**
   - 制度が存在するだけでなく、実際に運用されているか
   - 例：ローテーション制度があっても、実施人数が年1名では不十分

2. **定量と定性のバランス**
   - 数値だけでなく、人材の満足度・モチベーションも重要
   - 例：研修実施率100%でも、内容が形式的では意味がない

3. **長期視点の確認**
   - 人材育成は短期成果が出にくい
   - 3-5年の視点での計画性を評価

4. **ビジネス側の評価**
   - IT人材がビジネス部門からどう評価されているか
   - 「IT人材が頼りになる」という認識があるか
