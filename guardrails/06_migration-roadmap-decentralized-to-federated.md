# 分散型→連邦型 移行ロードマップ

## なぜこの移行が推奨されるか

### ビジネス背景
- M&Aや急成長により、各地域/事業部が独自にシステムを構築
- 重複投資、セキュリティリスク、データサイロ化が顕在化
- グローバル展開のため、最低限の統制が必要に

### 移行の優位性

- ✅ 既存の自律性を奪わないため、抵抗が少ない
- ✅ 「改善」として受け入れられやすい（権限縮小ではなく支援提供）
- ✅ コンプライアンス強化の名目で正当化しやすい
- ✅ クイックウィン（重複削減）が見えやすい

<br>

## 3フェーズ移行アプローチ

### Phase 1: 基盤整備

**目的**: 可視化と信頼構築

| アクション | 内容 | 責任者 |
|----------|------|--------|
| 現状調査 | 各ドメインのシステム/投資/リスク棚卸し | 中央IT + ドメインリーダー |
| Quick Win特定 | 重複投資/コンプライアンスギャップ抽出 | 中央IT |
| ガバナンス憲章策定 | Level 0文書作成、ステークホルダー合意 | CxO |
| 推進体制構築 | ドメインチャンピオン選出、委員会設置 | CxO |

**マイルストーン**: ガバナンス憲章承認、初回ガバナンス委員会開催

<br>

### Phase 2: コアガードレール導入

**目的**: 最重要領域から段階的に統制

#### 優先順位付け

**1st Wave: コンプライアンス・セキュリティ（必須）**
- GR-101: 認証・認可ガードレール
- GR-103: 脆弱性・パッチ管理ガードレール
- GR-106: インシデント対応ガードレール
- GR-109: 個人情報保護ガードレール

*理由*: 法規制対応、リスク低減で正当化しやすい

**2nd Wave: IT投資の可視化（Quick Win）**
- GR-501: IT投資意思決定ガードレール
- GR-504: クラウドFinOpsガードレール

*理由*: 重複削減効果が見えやすく、CFOの支持を得やすい

**3rd Wave: 技術標準化**
- GR-202: 技術スタック標準化ガードレール
- GR-201: クラウド戦略・選択ガードレール

*理由*: 既存の成功事例を標準化のベースとする

#### 各Waveの導入ステップ
```
1. ドラフト作成（中央IT）
2. ドメインレビュー
3. 調整・合意形成
4. 試行運用
5. フィードバック反映
6. 全社展開
```

**マイルストーン**: 各Wave完了時点で遵守率80%以上

<br>

### Phase 3: 全体最適化

**目的**: 残りのガードレール展開と継続改善

- 残りのガードレール展開（POL-200, 300, 400の詳細）
- 自動化・技術的強制の実装
- ドメイン間ベストプラクティス共有
- ガバナンス成熟度評価

**マイルストーン**: 全44ガードレール運用開始、遵守率95%

<br>

## 成功要因とリスク

### 成功要因

1. **地域チャンピオンの育成**
   - 各ドメインから推進者を選出
   - 中央と地域の橋渡し役

2. **既存の成功を標準に**
   - あるドメインの優れた取り組みを全社標準に
   - 「上から押し付け」ではなく「仲間の成功」として

3. **段階的エスカレーション**
   - 最初はグリーンゾーンを広く設定
   - 運用が安定したら徐々に厳格化

4. **可視化による説得**
   - 重複投資額、セキュリティリスク、コンプライアンス違反を数値化
   - データでガバナンス強化の必要性を示す

### リスクと対策

| リスク | 対策 |
|-------|------|
| ドメインの抵抗 | Quick Winを早期に示す、既存の成功を尊重 |
| 過度な官僚化 | グリーンゾーンを広く、承認期限を明確に |
| 中央の能力不足 | ドメインエキスパートを中央委員会に招聘 |
| 形骸化 | 四半期レビュー、遵守率測定、継続改善 |

## KPI設定

### プロセスKPI
- ガードレール遵守率: 95%以上（Phase 3終了時）
- 承認期限内処理率: 90%以上
- 例外申請承認率: 30%以下（適切な境界設定の証）

### 成果KPI
- IT投資の重複削減: 20%以上
- セキュリティインシデント: 前年比50%減
- 意思決定スピード: 維持または改善（グリーンゾーン案件）
- コンプライアンス監査指摘: 0件

<br>

## ガバナンス成熟度評価

各四半期で以下を評価：

| レベル | 状態 | 特徴 |
|-------|------|------|
| Level 1: 初期 | ガードレール未整備 | アドホックな判断 |
| Level 2: 定義済み | ガードレール文書化 | 文書はあるが遵守率低 |
| Level 3: 管理 | 遵守率80%以上 | プロセスが定着 |
| Level 4: 測定 | KPI測定・改善サイクル | データドリブンな改善 |
| Level 5: 最適化 | 自動化・技術的強制 | 継続的最適化 |

**目標**: Phase 3終了時点でLevel 4到達
