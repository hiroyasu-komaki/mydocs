# マチュリティ定義：OT/IT統合・プロセス安全

## 軸の定義

製造業における運用技術（OT: Operational Technology）とIT（Information Technology）の統合度合い、およびサイバーセキュリティとプロセス安全の統合管理レベルを評価する軸。
「IT vs OT」の分断から「IT×OT」の統合リスク管理と価値創出への変革度合いを測定する。

---

## レベル1：分離運用型（Isolated）

### 状態
- OTとITはほぼ完全に分離され、それぞれ独立して運用されている
- OT資産の可視化・管理が不十分
- サイバーセキュリティとプロセス安全の連携はほぼ存在しない

### 具体的な特徴

**資産管理：**
- OT資産（制御システム、PLC、HMI、センサー等）の台帳が存在しないか、Excelベースで手作業管理
- ネットワーク構成図が古い、または存在しない
- 資産の追加・変更が記録されていない

**ネットワーク・セキュリティ：**
- OTネットワークとITネットワークの分離が不明確、またはエアギャップのみ
- OT側のセキュリティ対策は最小限（アンチウイルスすら未導入の場合あり）
- OT環境へのアクセス管理が属人的（共有ID使用等）

**組織・プロセス：**
- OT担当（生産技術、保全等）とIT担当（情報システム）が別組織で、連携が弱い
- OT関連のインシデント対応手順が未整備
- プロセス安全評価（HAZOP、LOPA等）にサイバー脅威が考慮されていない

**リスク管理：**
- OTに対するサイバーリスク評価が実施されていない
- プロセス安全とサイバーセキュリティが別々に管理されている
- SIS（安全計装システム）のサイバー脆弱性が評価されていない

### チェック項目
- [ ] OT資産台帳が存在しない、または更新されていない（更新頻度年1回未満）
- [ ] OT/ITネットワーク間の境界防御（DMZ、ファイアウォール等）が未実装
- [ ] OTセキュリティインシデント対応手順書が存在しない
- [ ] プロセス安全担当とITセキュリティ担当の定例会議がない
- [ ] OTに対するサイバーリスク評価を過去3年実施していない

### 典型的な課題
- OT環境のサイバーリスクが不明
- インシデント発生時の対応混乱
- レガシーOTシステムの脆弱性放置
- 生産停止リスクの増大

## レベル2：基礎整備型（Managed）

### 状態
- OT資産の可視化が進み、基本的な台帳管理と定期更新が実施されている
- OT/IT間にDMZや境界防御が導入され、最低限の分離と監視を実施
- OT担当とIT担当の連携が開始されている

### 具体的な特徴

**資産管理：**
- 全工場のOT資産台帳が整備され、四半期ごとに更新されている
- 自動ディスカバリツールを導入し、未管理資産を定期的に検出
- 重要資産（SIS、DCS等）の優先的な管理

**ネットワーク・セキュリティ：**
- OT/IT間にDMZ（非武装地帯）を設置、境界ファイアウォールを実装
- 最低1工場でOTネットワーク可視化ツールが稼働
- OT環境へのアクセスログ取得開始

**組織・プロセス：**
- プロセス安全担当とITセキュリティ担当の月次定例会議を開催
- OTセキュリティインシデント対応手順書（ドラフト版）を作成
- 年1回の机上訓練を実施

**リスク管理：**
- OT資産の脆弱性を初期評価（ツールスキャンまたは手動チェック）
- OTリスクを全社リスクレジスタに反映開始
- HAZOPの一部案件でサイバー脅威シナリオを試験的に導入（1-2件）

### チェック項目
- [ ] 全工場のOT資産台帳が存在し、四半期ごとに更新されている
- [ ] 最低1工場でOT/ITネットワーク分離（DMZ等）が実装されている
- [ ] OT関連セキュリティインシデント対応手順書が存在し、年1回訓練実施
- [ ] プロセス安全担当とITセキュリティ担当の定例会議が月1回開催
- [ ] OTネットワーク可視化ツールが最低1工場で稼働

### 典型的な課題
- ツール導入後の運用定着
- OT/IT両部門の文化・用語の違い
- レガシーシステムへのセキュリティ対策の困難さ
- 予算・リソース不足

---

## レベル3：統合管理型（Integrated）

### 状態
- OT/IT共通のリスクアセスメントが全工場で定期実施され、統合的にリスク管理されている
- インシデント対応訓練が年1回以上実施され、継続的に改善されている
- スマートマニュファクチャリングPilotが稼働し、OT/IT統合の価値が実証されている

### 具体的な特徴

**資産管理：**
- 全工場のOT資産が一元管理され、リアルタイムまたは週次で更新
- 資産のライフサイクル管理（調達、導入、更新、廃棄）が標準化
- 構成管理データベース（CMDB）にOT資産が統合

**ネットワーク・セキュリティ：**
- 主要工場でOT/ITセグメンテーションが実装され、監視されている
- OTネットワークトラフィックの異常検知（IDS/IPS）を導入
- ゼロトラストの考え方を一部導入（最小権限アクセス等）

**組織・プロセス：**
- OT/IT統合インシデント対応訓練を年1回以上実施、訓練結果を手順書に反映
- プロセス安全評価（HAZOP/LOPA）にサイバー脅威が標準的に組み込まれている
- OT/IT横断のセキュリティチームが設置されている

**リスク管理：**
- IEC 62443（産業サイバーセキュリティ標準）に基づくリスクアセスメントを全工場で年1回実施
- プロセス安全とサイバーセキュリティのリスクを統合評価
- SISのサイバー脆弱性評価を定期実施（SIL評価時にサイバー脅威を考慮）

**価値創出：**
- 最低1工場でスマートマニュファクチャリングPilot（予知保全、ライン可視化等）が稼働
- Pilotの成果が定量評価され、ROIが明確

### チェック項目
- [ ] OT/IT共通リスクアセスメント（IEC 62443+プロセス安全）が全工場で年1回実施
- [ ] OT/IT統合インシデント対応訓練が年1回以上実施され、改善が反映されている
- [ ] 最低1工場でスマートマニュファクチャリングPilotが稼働し、成果が定量評価されている
- [ ] OT/ITセグメンテーションが主要工場で実装され、監視されている
- [ ] プロセス安全評価（HAZOP/LOPA）にサイバー脅威が標準組み込みされている

### 典型的な課題
- Pilotの横展開（コスト、リソース、技術的課題）
- レガシーOTシステムのセキュリティ対策
- OT/IT人材のスキルギャップ
- 運用負荷の増大

## レベル4：最適化型（Optimized）

### 状態
- OT/IT統合SOCで一元的に監視し、自動化された検知・対応フローが運用されている
- ゼロトラストに近いセグメンテーションと認証が実装されている
- スマートマニュファクチャリングが複数工場で展開され、継続的に改善されている

### 具体的な特徴

**資産管理：**
- OT資産がリアルタイムで自動更新され、異常（未承認の変更等）が即座に検知
- デジタルツイン（仮想モデル）を活用した資産管理

**ネットワーク・セキュリティ：**
- OT/IT統合SOC（Security Operations Center）で24/7監視
- 自動化された脅威検知・対応（SOAR: Security Orchestration, Automation and Response）
- ゼロトラストアーキテクチャ（継続的認証、最小権限アクセス、マイクロセグメンテーション）

**組織・プロセス：**
- OT/ITインシデント対応が高度に自動化され、MTTD（平均検知時間）・MTTR（平均復旧時間）が継続的に改善
- クロスファンクショナルチーム（OT、IT、プロセス安全、品質等）が常設
- 定期的なレッドチーム演習（攻撃シミュレーション）を実施

**リスク管理：**
- リスク評価が定量的（確率×影響度の数値化）で、投資判断に活用
- サイバー保険の適用範囲にOTを含む
- 継続的な脆弱性管理（パッチ適用、代替対策）

**価値創出：**
- スマートマニュファクチャリングが複数工場で展開
- OT/IT統合によるビジネス価値（生産性向上、品質改善等）が定量的に示されている

### チェック項目
- [ ] OT/IT統合SOCで24/7監視を実施
- [ ] 自動化された脅威検知・対応フローが運用されている
- [ ] ゼロトラストアーキテクチャの主要要素が実装されている
- [ ] スマートマニュファクチャリングが複数工場で展開され、ROIが測定されている
- [ ] MTTD・MTTRが四半期ごとに測定され、改善されている

### 典型的な課題
- 高度な自動化の維持・運用コスト
- 誤検知の削減
- 人材の高度化継続
- ビジネス環境変化への適応

## レベル5：イノベーション型（Innovating）

### 状態
- 経営レベルのサイバーリスク管理（ガバナンス）にOTリスクが完全に統合されている
- OT/IT統合が競争優位の源泉として機能（業界をリードする存在）
- 新技術（AI、エッジコンピューティング等）を積極的に導入し、継続的にイノベーション

### 具体的な特徴

**資産管理：**
- AIを活用した予測的資産管理（故障予測、最適更新タイミング等）

**ネットワーク・セキュリティ：**
- AIベースの異常検知（機械学習による正常パターン学習、異常検知）
- 量子暗号など最先端技術の導入検討・実装

**組織・プロセス：**
- OT/ITの区別が曖昧になり、完全に統合された組織・プロセス
- サイバーレジリエンス（回復力）の考え方が浸透し、インシデント前提の設計

**リスク管理：**
- 取締役会レベルでOTサイバーリスクが定期的に報告・議論されている
- リスクベースの投資判断が高度化（シミュレーション、シナリオ分析等）

**価値創出：**
- OT/IT統合によるイノベーション（新製品、新サービス、新ビジネスモデル）
- 業界標準化活動への参画、知見の外部発信

### チェック項目
- [ ] 取締役会でOTサイバーリスクが四半期ごとに報告されている
- [ ] AIベースの異常検知・予測的管理が実装されている
- [ ] OT/IT統合によるイノベーション事例が年1件以上ある
- [ ] 業界標準化活動やカンファレンスで知見を発信している

### 典型的な課題
- イノベーションと安定運用のバランス
- 規制対応（新技術の承認取得）
- 組織文化の継続的変革

## レベル間の移行に必要な主要アクション

### 1→2への移行
- OT資産棚卸しの実施（全工場）
- DMZ・境界防御の導入（最低1工場で先行）
- OT/IT定例会議の設置

### 2→3への移行
- 全工場でのリスクアセスメント実施（IEC 62443ベース）
- インシデント対応訓練の定例化
- スマートマニュファクチャリングPilot実施

### 3→4への移行
- OT/IT統合SOCの構築
- 自動化された検知・対応フローの実装
- スマートマニュファクチャリングの横展開

### 4→5への移行
- 経営レベルのOTリスクガバナンス統合
- AI等の新技術導入
- 業界リーダーシップの確立

## 評価時の留意点

1. **製造業特有の制約理解**
   - OT環境は「止められない」「パッチ適用困難」等の制約がある
   - IT標準のセキュリティ対策がそのまま適用できない場合が多い

2. **プロセス安全との統合確認**
   - サイバーセキュリティだけでなく、プロセス安全（SIL、HAZOP等）との統合が重要
   - 「安全性を損なわずにセキュリティを高める」視点

3. **工場間のばらつき確認**
   - 「一部の工場だけ」でなく、全工場での実施状況を確認
   - 重要度に応じた優先順位付けは許容

4. **OT/IT両部門の評価**
   - IT部門だけでなく、OT担当（生産技術、保全等）からも評価を聴取
   - 両部門の認識ギャップに注意
