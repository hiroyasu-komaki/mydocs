# Year 2：重点軸でレベル2達成（基盤の実装）

## 年間目標
- 重点3軸（OT/IT統合、人材・組織、ITガバナンス）をレベル2.0にクリア
- モニタリングサイクルを4回転させ、改善PDCAを定着
- Year 3への準備（深堀り軸の選定）

## 重点3軸の選定理由
- **OT/IT統合**：製造業の差別化領域、Year 3レベル3の前提
- **人材・組織**：全軸の前提条件、ここが弱いと他が進まない
- **ITガバナンス**：OT/IT統合レベル3の前提、セキュリティ基盤必須

---

## 軸1：OT/IT統合・プロセス安全

### レベル2チェックリスト

- [ ] 全工場のOT資産台帳が存在し、四半期ごとに更新されている
- [ ] 最低1工場でOT/ITネットワーク分離（DMZ等）が実装されている
- [ ] OT関連セキュリティインシデント対応手順書が存在し、年1回訓練実施
- [ ] プロセス安全担当とITセキュリティ担当の定例会議が月1回開催
- [ ] OTネットワーク可視化ツールが最低1工場で稼働

### 主要イニシアチブ

#### 2.1 OT資産管理の全社展開（6-9ヶ月）

**実施内容：**
- 全工場での資産台帳整備（Year 1パイロットを横展開）
- 自動ディスカバリツール導入（1工場で先行）

**マイルストーン：**
- Q1：ツール選定、導入計画策定
- Q2：1工場でツール導入、運用開始
- Q3：他工場への展開開始
- Q4：全工場でベースライン完成

**KPI：**
- OT資産台帳登録率：60% → 95%

**リソース：**
- IT/OTセキュリティ：フルタイム1-2名×12ヶ月
- 外部コンサル：月2-3日×6ヶ月
- 現場エンジニア：各工場で月2-3日

#### 2.2 セキュリティと安全の連携開始（通年）

**実施内容：**
- プロセス安全側のHAZOP/LOPAにサイバー脅威シナリオを試験導入（1-2案件）
- プロセス安全エンジニアとITセキュリティの月次定例会設置
- 「サイバー攻撃によるSIS機能喪失」シナリオの評価を1件実施

**定例会議アジェンダ例：**
1. 前月のOT/ITインシデント共有
2. 新規プロジェクトのリスク確認
3. サイバー×安全の統合課題議論
4. 次月のアクション確認

**KPI：**
- 定例会議開催回数：年12回
- HAZOPでサイバーシナリオを扱った案件数：1-2件

#### 2.3 OT/ITインシデント対応体制の構築（6ヶ月）

**実施内容：**
- OT/IT統合インシデント対応手順書の作成
- 机上演習を年1回実施（Year 2終盤）

**手順書の構成：**
1. インシデント分類（IT/OT/統合）
2. エスカレーションフロー
3. 関係者連絡先
4. 初動対応チェックリスト
5. 復旧手順（IT/OT別）

**KPI：**
- インシデント対応訓練実施回数：0回 → 1回

## 軸2：人材・組織能力

### レベル2チェックリスト

- [ ] 主要IT役割のスキル定義書が存在する
- [ ] 各役割の年間研修計画が存在し、実施記録がある
- [ ] ジョブローテーションが過去1年で最低2件実施されている
- [ ] IT人材の評価基準が文書化され、運用されている
- [ ] 外部依存度の方針が明文化されている

### 主要イニシアチブ

#### 2.4 スキル定義と育成体系の整備（6ヶ月）

**対象役割（5-6個）：**
1. IT企画・アーキテクト
2. アプリケーション開発
3. インフラ・運用
4. セキュリティ
5. OT-ITブリッジ
6. ビジネスアナリスト

**各役割のスキル定義項目：**
- 必須スキル（技術、ビジネス、ソフトスキル）
- レベル別要件（初級/中級/上級）
- 推奨資格
- 育成パス

**研修プログラム：**
- 社内研修（OJT、メンタリング）
- 外部研修（ベンダー、専門機関）
- e-learning、資格取得支援

**KPI：**
- スキル定義済み役割比率：0% → 100%
- 研修実施率：目標80%以上

#### 2.5 ジョブローテーションの試行（通年）

**Year 2の実施規模：**
- ITから製造現場へ：1-2名、3-6ヶ月
- ビジネス部門からIT部門へ：1名、3-6ヶ月

**ローテーション設計：**
- 目的：相互理解、橋渡し人材育成
- 期間：3-6ヶ月（プロジェクトベース）
- 評価：ローテーション前後のスキル変化、橋渡し成果

**ナレッジ共有：**
- 四半期ごとに経験者の共有会開催
- 学びのドキュメント化（Wiki等）

**KPI：**
- ジョブローテーション参加者数：0名 → 3名

#### 2.6 外部依存度の見直し（3-4ヶ月）

**実施内容：**
- コア領域の定義（アーキテクチャ、要件定義、OT/IT連携等）
- 外部委託の判断基準策定
- 3年後の内製化目標設定

**判断基準の3軸：**
1. 競争優位に直結するか
2. 長期的に継続する能力か
3. 市場にスキルが潤沢か

**KPI：**
- 外部委託費比率（ベースライン把握、Year 3で削減目標設定）

## 軸3：ITガバナンス・セキュリティ

### レベル2チェックリスト

- [ ] 変更管理プロセスが文書化され、年1回以上実行
- [ ] IT投資評価プロセスが存在し、案件起案時に運用
- [ ] リスク管理プロセス（IT+OT）が文書化され、年1回評価実施
- [ ] セキュリティポリシーがOT環境を考慮して更新
- [ ] 主要ベンダーのSLAが整備され、定期レビューされている

### 主要イニシアチブ

#### 2.7 基本プロセスの文書化と実行（6-9ヶ月）

**対象プロセス：**
1. 変更管理（Change Management）
2. IT投資評価（Investment Evaluation）
3. リスク管理（Risk Management）

**各プロセスの文書化内容：**
- 目的、適用範囲
- プロセスフロー図
- 責任者・関係者
- 実施頻度
- 成果物・記録

**実行とレビュー：**
- 各プロセスを最低年1回実行
- 実行後に改善点を記録
- 次回実行時に反映

**KPI：**
- 文書化済みプロセス数：0-1 → 3-5
- 実行回数：各プロセス年1回以上

#### 2.8 OT/ITリスク統合管理の開始（6ヶ月）

**実施内容：**
- IT/OT共通リスクレジスタの作成
- リスクアセスメントを年1回実施（IEC 62443ベース）

**リスクレジスタの項目：**
- リスクID、分類（IT/OT/統合）
- 脅威シナリオ
- 影響度、発生確率
- 対策状況、責任者

**KPI：**
- OTリスクのレジスタ反映率：0% → 100%

#### 2.9 ベンダー管理の基盤整備（3-4ヶ月）

**実施内容：**
- 主要ベンダー（5-10社）のSLA整備
- ベンダー評価を年1回実施

**SLA項目例：**
- サービス範囲
- 品質基準（可用性、応答時間等）
- レビュー頻度
- ペナルティ条項

**KPI：**
- SLA整備済みベンダー比率：目標80%以上

### 未達時の対応方針

**レベル2未達の軸が2つ以上：**
- Year 3の深堀り計画を1年延期
- レベル2達成に集中

**人材・組織がレベル2未達：**
- 最優先で対処（他軸の前提条件のため）

**1軸のみ未達：**
- Year 3で並行して完了させる
