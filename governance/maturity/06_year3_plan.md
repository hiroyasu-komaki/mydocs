# Year 3：選択軸でレベル3達成（組織的定着）

## 年間目標
- 選択3軸（ビジネス連携、OT/IT統合、人材・組織）をレベル3.0に到達
- スマートマニュファクチャリングPilotで見える成果を創出
- 3年間の振り返りとYear 4以降への継承

## 選択3軸と非選択軸

### 選択軸（深堀り）
1. **ビジネス連携・価値創出** - DXビジョンの中核
2. **OT/IT統合・プロセス安全** - 製造業の差別化
3. **人材・組織能力** - 他軸の前提

### 非選択軸（維持）
- ベンダー共創、ITガバナンス、デジタル化：**レベル2.0を維持**
- 深堀りはしないが、四半期レビューで退行確認

---

## 軸1：ビジネス連携・価値創出

### レベル3チェックリスト

- [ ] 中期経営計画と整合したIT/DXロードマップが存在し、年次更新
- [ ] IT投資案件の70%以上が起案時にビジネスKPIを明記
- [ ] 投資後1年以内の効果検証が標準化され、実施率80%以上
- [ ] 部門横断テーマで共通KPIが設定されている
- [ ] IT投資ポートフォリオ全体のROIが年次で評価されている

### 主要イニシアチブ

#### 3.1 IT/DXロードマップの策定（3-4ヶ月）

**実施内容：**
- 経営企画、主要事業部門とのワークショップ（年2-3回）
- 事業戦略と整合した3年IT計画の作成

**ワークショップ構成：**
1. 中期経営計画の確認
2. 各事業部門の課題・ニーズ抽出
3. IT/DX施策の優先順位付け
4. 3年ロードマップドラフト作成
5. 経営層レビューと承認

**成果物：**
- IT/DXロードマップ（3年、年次更新）
- 部門別IT施策一覧
- 投資計画（概算）

**KPI：**
- ビジネス部門との共同策定セッション回数：年2-3回

#### 3.2 投資評価・効果検証の標準化（通年）

**実施内容：**
- 案件起案テンプレートにビジネスKPI欄を必須化
- 投資後レビューを四半期実施（着手1年経過案件）

**起案テンプレート項目：**
- ビジネス目標・KPI（必須）
- 投資金額・期間
- 期待効果（定量・定性）
- リスク・前提条件
- 効果検証方法・タイミング

**効果検証プロセス：**
1. 投資後6ヶ月：中間レビュー
2. 投資後12ヶ月：最終評価
3. KPI達成状況の記録
4. 学びの抽出と次案件への反映

**KPI：**
- ビジネスKPI明記案件比率：30% → 70%
- 投資後効果検証実施率：50% → 80%

#### 3.3 ポートフォリオ管理の導入（6ヶ月）

**実施内容：**
- IT投資案件をカテゴリ分類
- ポートフォリオ全体のROI・KPI達成率を年次評価

**カテゴリ分類例：**
- 守り（インフラ維持、セキュリティ）vs 攻め（新事業、効率化）
- 戦略（競争優位）vs オペレーション（業務改善）

**ポートフォリオ評価項目：**
- カテゴリ別投資比率
- 全体ROI
- KPI達成率
- リスク分布

**KPI：**
- ポートフォリオROI（Year 3終盤に初回測定）

**リソース：**
- IT企画：フルタイム2名×12ヶ月
- 事業部門キーユーザー：各部門1名が月3-5日
- 外部コンサル：月2日×4ヶ月

## 軸2：OT/IT統合・プロセス安全

### レベル3チェックリスト

- [ ] OT/IT共通リスクアセスメント（IEC 62443+プロセス安全）が全工場で年1回実施
- [ ] OT/IT統合インシデント対応訓練が年1回以上実施され、改善反映
- [ ] 最低1工場でスマートマニュファクチャリングPilotが稼働し、成果が定量評価
- [ ] OT/ITセグメンテーションが主要工場で実装され、監視されている
- [ ] プロセス安全評価（HAZOP/LOPA）にサイバー脅威が標準組み込み

### 主要イニシアチブ

#### 3.4 OT/IT統合リスク管理の全社展開（6-9ヶ月）

**実施内容：**
- 全工場でOT/IT共通リスクアセスメントを実施
- リスク対策の優先順位付けと実行計画策定

**アセスメントプロセス：**
1. 各工場の資産・ネットワーク確認
2. 脅威シナリオの特定
3. 影響度・発生確率の評価
4. 対策の優先順位付け
5. 実行計画策定

**KPI：**
- リスクアセスメント実施工場比率：33% → 100%

#### 3.5 インシデント対応能力の向上（通年）

**実施内容：**
- OT/IT統合インシデント対応訓練を年1回実施
- 訓練結果を手順書に反映し、改善PDCAを回す

**訓練シナリオ例：**
- ランサムウェア攻撃による生産停止
- 制御システムへの不正アクセス
- OTネットワークからのデータ流出

**KPI：**
- 訓練実施回数：年1回
- 改善項目の反映率：100%

#### 3.6 スマートマニュファクチャリングPilot（9-12ヶ月）

**Pilot候補：**

1. **予知保全Pilot**
   - 製造設備の振動・温度データから故障予測
   - 目標：ダウンタイム20-30%削減
   
2. **ライン可視化Pilot**
   - リアルタイム生産モニタリング
   - 目標：ボトルネック特定、OEE向上
   
3. **品質予測Pilot**
   - 工程データと品質相関分析
   - 目標：不良率削減、歩留まり改善

**選定基準：**
- 投資規模が小さい（数百万～数千万円）
- 効果測定が容易
- 横展開可能性が高い

**実装ステップ：**
1. Pilot案件選定（Q1）
2. システム設計・構築（Q2-Q3）
3. 試験運用・調整（Q3-Q4）
4. 成果評価・横展開計画（Q4）

**KPI：**
- Pilot成果（ダウンタイム削減率、ROI等）

#### 3.7 プロセス安全へのサイバー統合（通年）

**実施内容：**
- HAZOP/LOPAの標準手順にサイバー脅威評価を組み込み
- プロセス安全エンジニアへのサイバー研修実施

**KPI：**
- サイバーシナリオ評価済みHAZOP比率：目標50%以上

**リソース：**
- IT/OTセキュリティ：フルタイム2名×12ヶ月
- 現場エンジニア：各工場で月5-10日
- スマートファクトリーPilot：外部ベンダー支援＋機器投資
- 外部コンサル：月3-4日×6ヶ月

## 軸3：人材・組織能力

### レベル3チェックリスト

- [ ] キャリアパス・ローテーション制度が正式化され、運用されている
- [ ] ITリーダーの30%以上がビジネス部門経験を持つ
- [ ] ローテーション経験者のナレッジベースが整備され、活用されている
- [ ] IT人材ポートフォリオが計画的に育成されている
- [ ] チームパフォーマンスの定量モニタリングが開始されている

### 主要イニシアチブ

#### 3.8 ジョブローテーション制度の正式化（3-4ヶ月）

**制度設計項目：**
- 対象者（ITリーダー候補、中堅層）
- 期間（3-6ヶ月）
- 受け入れ部門の選定基準
- 評価方法（ローテーション前後のスキル変化）
- 復帰後の処遇（昇進・昇格への反映）

**経営層承認：**
- 制度案の作成
- 人事部門との調整
- 経営会議での承認取得

**KPI：**
- 制度化完了
- 年間ローテーション目標人数設定

#### 3.9 ローテーションの拡大実施（通年）

**Year 3の実施規模：**
- 年間5名以上のローテーション実施

**実施パターン：**
- IT → 製造：2名
- IT → 営業/R&D：2名
- ビジネス → IT：1名

**ナレッジ共有：**
- 四半期ごとにナレッジ共有会開催
- ローテーション経験者がメンター役を担う

**KPI：**
- ジョブローテーション実施人数：3名 → 5名以上

#### 3.10 ナレッジ定着の仕組み化（6ヶ月）

**実施内容：**
- ナレッジベース構築（Wiki、事例集等）
- ローテーション経験者の「橋渡し機能」を評価指標化

**ナレッジベース構成：**
- ローテーション体験記
- 部門間の違い（文化、用語、プロセス）
- 橋渡しのコツ
- よくある課題と対処法

**KPI：**
- ナレッジベース登録件数：目標20件以上
- 活用頻度：月間アクセス数

#### 3.11 IT人材ポートフォリオの計画的育成（通年）

**実施内容：**
- 将来必要な役割を定義（データサイエンティスト、AIエンジニア等）
- 現状とのギャップ分析、育成計画策定
- 外部採用 vs 内部育成の判断

**育成計画：**
- 3年後の目標人員構成
- 役割別の育成パス
- 外部採用が必要な役割

**KPI：**
- 計画的育成対象役割の充足率

**リソース：**
- 人事/IT企画：フルタイム1名×12ヶ月
- ローテーション参加者：5名×3-6ヶ月
- 外部コンサル：月1日×3ヶ月
