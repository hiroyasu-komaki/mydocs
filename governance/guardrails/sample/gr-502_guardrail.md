# ガードレール文書

## GR-502: ビジネスケース・ベネフィット測定ガードレール

### 1. メタデータ

| 項目 | 内容 |
|-----|------|
| **ガードレールID** | GR-502 |
| **対応ポリシー** | POL-500: IT投資・ポートフォリオ管理ポリシー |
| **バージョン** | 1.0 |
| **発行日** | 2025-02-08 |
| **次回レビュー** | 2025-08-08（半期） |
| **承認者** | IT投資委員会 |
| **所管** | IT企画部門 |

### 2. 目的とスコープ

**目的:**
IT投資のビジネスケース作成基準を標準化し、投資効果を継続的に測定・検証するフレームワークを確立する。

**スコープ:**
- ビジネスケース作成要件（簡易版・フル版）
- 財務分析手法と基準
- ベネフィット測定タイミングと方法
- 未達成時の対応プロセス

<br>

#### 3.4 財務分析手法

##### 3.4.1 投資収益率（ROI）

**計算式:**
```
ROI (%) = (累積利益 - 投資額) / 投資額 × 100
```

**期間:** 3年間（標準）、5年間（大型インフラ投資）

**推奨基準:**
- 3年ROI ≥ 100%（中型投資）
- 3年ROI ≥ 150%（大型投資）

**例:**
```
投資額: 1,000万円
年間利益: 500万円（1年目）、600万円（2年目）、700万円（3年目）
累積利益: 1,800万円
ROI = (1,800 - 1,000) / 1,000 × 100 = 80%

→ 基準未達のため、追加ベネフィットの検討または却下
```

##### 3.4.2 正味現在価値（NPV）

**計算式:**
```
NPV = Σ(CFt / (1 + r)^t) - 初期投資額
CFt: t年目のキャッシュフロー
r: 割引率（標準8%）
```

**判定基準:**
- NPV > 0: 承認可
- NPV < 0: 却下

**割引率:**
- 標準: 8%（WACCベース）
- 高リスク投資（Transform）: 10%
- 低リスク投資（Run）: 6%

##### 3.4.3 投資回収期間（Payback Period）

**計算式:**
```
回収期間 = 初期投資額 / 年間平均キャッシュフロー
```

**推奨基準:**
- 中型投資: 3年以内
- 大型投資: 3年以内（推奨）、最大5年
- Run投資: 2年以内

##### 3.4.4 感度分析（大型投資必須）

以下の変数について±20%の感度分析を実施:
- ベネフィット額
- 投資額
- 導入期間
- 利用率

**アウトプット:**
- 各変数の影響度ランキング
- 最悪ケースでのROI/NPV

#### 3.5 ベネフィット分類

##### 3.5.1 定量的ベネフィット（測定必須）

| カテゴリ | 例 | 測定方法 |
|---------|---|---------|
| **コスト削減** | 人件費削減、運用コスト削減、ライセンス削減 | Before/After比較 |
| **収益増加** | 売上増、新規顧客獲得 | 財務データ |
| **生産性向上** | 処理時間短縮、エラー削減 | 工数測定、エラー率 |
| **コスト回避** | 障害防止、罰金回避 | リスク分析 |

**測定タイミング:**
- ベースライン: 導入前3ヶ月
- 導入直後: 導入後1ヶ月
- 安定期: 導入後6ヶ月、12ヶ月

##### 3.5.2 定性的ベネフィット（記述必須）

| カテゴリ | 例 | 評価方法 |
|---------|---|---------|
| **顧客満足度** | サービス品質向上、応答時間短縮 | NPS, CSAT調査 |
| **従業員満足度** | 業務負荷軽減、スキル向上 | 従業員サーベイ |
| **リスク低減** | セキュリティ強化、コンプライアンス改善 | リスク評価 |
| **戦略的価値** | 市場競争力、イノベーション促進 | 定性評価 |

**評価タイミング:** 導入後6ヶ月、12ヶ月

## ガードレール

### 4. 自律判断の境界（グリーンゾーン）

#### 4.1 簡易版ビジネスケースで判断可能

**条件:**
- 中型投資（100万〜1,000万円）
- 定量的ベネフィットが明確（ROI計算可能）
- 既知技術の活用
- リスクが「低」または「中」

**必要なアクション:**
- 簡易版ビジネスケース作成（1-2ページ）
- CTO/CFOへの提出
- 承認後の四半期レビュー（簡易版）

**判断者:** CTO + CFO

**承認期限:** 2週間以内

**例:**
- 既存SaaSの機能拡張（300万円）
- 部門内ツールの刷新（500万円）

### 5. 承認必要な範囲（イエローゾーン）

#### 5.1 フルビジネスケース必須

**条件（いずれか該当）:**
- 大型投資（1,000万円以上）
- 定量的ベネフィットの算出が困難
- 新技術・未経験領域への投資
- 複数年にわたる段階的投資
- リスクが「高」

**必要なアクション:**
- フルビジネスケース作成（5-10ページ）
- IT投資委員会での承認プレゼンテーション
- 承認後の月次進捗レビュー
- ベネフィット実現計画の提出

**判断者:** IT投資委員会

**承認期限:** 次回委員会（月次、最大1ヶ月）

**例:**
- 全社共通基盤の構築（3,000万円）
- 新規事業向けシステム開発（5,000万円）

#### 5.2 外部専門家レビュー推奨

**条件:**
- 投資額5,000万円以上
- 財務分析に不確実性が高い
- 複雑なM&Aに伴うシステム統合

**必要なアクション:**
- 外部コンサルタント/会計士によるレビュー
- レビューレポートの委員会提出

**費用:** プロジェクト予算に計上

### 6. 禁止・エスカレーション（レッドゾーン）

#### 6.1 禁止事項

| 禁止事項 | 理由 | 違反時の措置 |
|---------|------|-------------|
| ビジネスケースなしの中型以上投資 | 投資根拠不明 | 投資即時停止 |
| 虚偽または誇張されたベネフィット記載 | 意思決定歪曲 | 懲戒、プロジェクト中止 |
| 感度分析の省略（大型投資） | リスク不透明 | 申請差し戻し |
| 代替案検討の欠如 | 最適解未検証 | 申請差し戻し |

#### 6.2 即時エスカレーション事項

以下は発見次第CTOへエスカレーション:
- ベネフィット未達が30%以上見込まれる
- ビジネスケースの前提条件が大きく崩れた
- 投資額が当初計画より20%以上増加見込み

## ガードレール運用

### 7. 例外プロセス

#### 7.1 ベネフィット測定困難な投資

**適用ケース:**
- セキュリティ強化（損失回避型）
- コンプライアンス対応（法令遵守型）
- 技術的負債解消（将来リスク回避型）

**代替評価方法:**
- リスク低減効果の定量化（損失回避額）
- 罰金・訴訟リスクの定量化
- 将来コスト増加の回避額

**承認基準:**
- リスク評価が「高」以上
- 法令違反リスクあり
- 技術的負債が事業継続に影響

#### 7.2 段階的投資（Phased Investment）

**適用条件:**
- 総額1,000万円以上の投資を複数年で実施
- 各フェーズで価値検証が可能

**ビジネスケース要件:**
- 全体ビジネスケース（総額ベース）
- 各フェーズのマイルストーンとゲート基準
- フェーズ毎の投資判断基準（Go/No-Go）

**承認プロセス:**
- 全体計画の承認（IT投資委員会）
- 各フェーズ開始時の再承認（簡易審査）

### 8. ベネフィット測定プロセス

#### 8.1 測定タイミング

| タイミング | 目的 | 測定内容 | 報告先 |
|-----------|------|---------|-------|
| **ベースライン** | 導入前の現状把握 | Before指標の測定 | プロジェクト記録 |
| **導入直後** | 初期効果確認 | 1ヶ月後の指標 | プロジェクトマネージャー |
| **3ヶ月後** | 早期効果確認 | 主要指標の測定 | ドメインリーダー |
| **6ヶ月後** | 中間評価 | 全指標の測定、ギャップ分析 | CTO（中型以上） |
| **12ヶ月後** | 年次評価 | 全指標の測定、ROI再計算 | IT投資委員会（大型） |
| **2年目以降** | 継続評価 | 年次測定（大型投資のみ） | IT投資委員会 |

#### 8.2 ベネフィット実現ギャップ対応

**ギャップ判定基準:**
- **軽微**: 計画の90%以上達成 → 経過観察
- **中程度**: 計画の70-90%達成 → 改善計画必須
- **重大**: 計画の70%未満 → 即時対応・エスカレーション

**対応プロセス:**

```
[ギャップ検出] → [原因分析] → [改善計画策定] → [承認] → [実行] → [再測定]
```

**改善計画の内容:**
- 未達成の根本原因分析
- 具体的な改善アクション
- 責任者とタイムライン
- 達成見込みの再評価

**承認者:**
- 中程度ギャップ: ドメインリーダー
- 重大ギャップ: CTO（中型）、IT投資委員会（大型）

#### 8.3 ポスト実装レビュー（PIR）

**実施タイミング:** プロジェクト完了後6ヶ月

**目的:**
- ビジネスケースとの実績比較
- 成功要因・失敗要因の抽出
- 組織的学習の促進

**参加者:**
- プロジェクトマネージャー
- ドメインリーダー
- 主要ステークホルダー
- 中央PMO（ファシリテーター）

**アジェンダ:**
1. プロジェクト概要の振り返り
2. ベネフィット実現状況（vs 計画）
3. 投資額・スケジュール実績（vs 計画）
4. 成功要因・改善点
5. 教訓の抽出
6. 今後のアクション

**アウトプット:**
- PIRレポート（2-3ページ）
- 教訓データベースへの登録
- 次期投資への反映事項

### 9. モニタリングと報告

#### 9.1 ベネフィット実現ダッシュボード

中央PMOが提供するダッシュボード項目:
- プロジェクト別ベネフィット実現率
- 計画vs実績（投資額、ROI、効果額）
- ギャップ分析（ヒートマップ）
- カテゴリ別実現率（Run/Grow/Transform）

**更新頻度:** 月次

**アクセス権限:**
- IT投資委員会: 全プロジェクト
- ドメインリーダー: 自ドメインプロジェクト
- プロジェクトマネージャー: 担当プロジェクト

#### 9.2 報告義務

| 投資分類 | 報告内容 | 報告タイミング | 報告先 |
|---------|---------|---------------|-------|
| 中型 | ベネフィット実現状況（簡易） | 6ヶ月後、12ヶ月後 | CTO |
| 大型 | ベネフィット実現状況（詳細） | 3ヶ月、6ヶ月、12ヶ月、24ヶ月 | IT投資委員会 |
| 戦略的 | ベネフィット実現状況（詳細）+ PIR | 同上 + PIR | IT投資委員会+経営会議 |

#### 9.3 KPI

- **ベネフィット実現率**: 目標80%以上
  - 計算: (実現ベネフィット / 計画ベネフィット) × 100%
  - 測定: 12ヶ月後
  
- **ビジネスケース承認率**: 70-80%（適切な審査の証）
  - 計算: (承認件数 / 申請件数) × 100%
  
- **PIR実施率**: 100%（大型投資）
  - 計算: (PIR実施件数 / 完了プロジェクト件数) × 100%
