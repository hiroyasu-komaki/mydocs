# ガードレール文書

## GR-501: IT投資意思決定ガードレール

### 1. メタデータ

| 項目 | 内容 |
|-----|------|
| **ガードレールID** | GR-501 |
| **対応ポリシー** | POL-500: IT投資・ポートフォリオ管理ポリシー |
| **バージョン** | 1.0 |
| **発行日** | 2025-02-08 |
| **次回レビュー** | 2025-08-08（半期） |
| **承認者** | IT投資委員会 |
| **所管** | IT企画部門 |

### 2. 目的とスコープ

**目的:**
IT投資の意思決定プロセスを標準化し、投資規模・影響範囲に応じた適切な承認レベルを定義する。ドメインの自律性を尊重しつつ、全社最適を実現する。

**スコープ:**
- すべてのIT関連投資の分類・評価・承認
- 中央とドメインの意思決定境界の明確化
- 投資優先順位付けプロセス

## スタンダード

### 3. 具体的基準

#### 3.1 投資カテゴリ定義

| カテゴリ | 定義 | 目的 | 例 |
|---------|------|------|---|
| **Run（維持）** | 既存システムの維持・運用に必要な投資 | 事業継続、品質維持 | インフラ保守、ライセンス更新、障害対応、セキュリティパッチ |
| **Grow（成長）** | 既存事業の拡大・効率化のための投資 | 収益拡大、コスト削減 | プロセス改善、既存システム機能追加、分析基盤強化 |
| **Transform（変革）** | 新規ビジネスモデル・イノベーションのための投資 | 競争優位性確立、市場創造 | 新規プラットフォーム、AI/ML活用、DX施策 |

**ポートフォリオバランス目標:**
- Run: 50-60%
- Grow: 25-35%
- Transform: 10-20%

#### 3.2 投資規模分類

| 分類 | 金額範囲（税抜） | 承認レベル | 備考 |
|------|-----------------|-----------|------|
| **小型** | 100万円未満 | ドメインリーダー | 簡易プロセス |
| **中型** | 100万円〜1,000万円 | CTO + CFO | ビジネスケース（簡易版）必須 |
| **大型** | 1,000万円以上 | IT投資委員会 | フルビジネスケース必須 |
| **戦略的** | 5,000万円以上 | IT投資委員会 + 経営会議 | 経営戦略との整合性審査 |

**注意:**
- 複数年契約の場合は総額で評価
- 関連投資の分割発注は禁止（年度内で合算評価）
- クラウドサービスは年間推定コストで評価

#### 3.3 評価基準

##### 3.3.1 定量評価（大型・中型投資必須）

| 指標 | 定義 | 推奨基準 | 必須レベル |
|-----|------|---------|-----------|
| **ROI** | (累積利益 - 投資額) / 投資額 × 100% | 3年で100%以上 | 中型以上 |
| **投資回収期間** | 初期投資を回収するまでの期間 | 3年以内 | 中型以上 |
| **NPV** | 正味現在価値（割引率: 8%） | 正の値 | 大型以上 |
| **コスト削減額** | 年間削減コスト（Run/Grow投資） | 定量化必須 | Run投資 |

##### 3.3.2 定性評価（全投資）

| 評価項目 | 評価基準 | スコア範囲 |
|---------|---------|-----------|
| **戦略適合度** | 経営戦略・IT戦略との整合性 | 1-5点 |
| **緊急性** | 実施タイミングの重要性 | 1-5点 |
| **実現可能性** | 技術的・組織的実現可能性 | 1-5点 |
| **リスク** | プロジェクト実行リスク | 1-5点（低リスク=5） |

##### 3.3.3 総合スコアリング（大型投資）

| 評価項目 | 配点 | 評価方法 |
|---------|------|---------|
| **ビジネス価値** | 40点 | ROI、売上/コスト影響 |
| **戦略適合度** | 30点 | 経営戦略との整合性（5段階×6点） |
| **リスク** | 20点 | 技術・ビジネスリスク評価（5段階×4点） |
| **実現可能性** | 10点 | スキル、リソース、期間（5段階×2点） |
| **合計** | 100点 | - |

**判定基準:**
- **70点以上**: 承認推奨
- **50-70点**: 条件付き承認（リスク緩和策必須）
- **50点未満**: 却下または再検討

## ガードレール

### 4. 自律判断の境界（グリーンゾーン）

#### 4.1 ドメイン完全自律判断可能

**条件（すべて満たす必要あり）:**
1. 投資額が100万円未満
2. ドメインの既承認予算内
3. 既存システムの改善・保守（新規構築でない）
4. 承認済み技術スタック使用
5. 他ドメインへの影響なし
6. セキュリティ・コンプライアンスリスクなし

**必要なアクション:**
- 投資台帳への登録（1週間以内）
- 四半期レポートでの事後報告

**判断者:** ドメインリーダー

**判断期限:** 即日〜3営業日

**プロセス:**
```
[案件発生] → [ドメインリーダー承認] → [投資台帳登録] → [実行] → [四半期報告]
```

**例:**
- 既存SaaSのライセンス追加（50万円）
- 開発環境のサーバー増強（80万円）
- 承認済みツールの追加購入（30万円）

### 5. 承認必要な範囲（イエローゾーン）

#### 5.1 中央技術評価必要（新技術・中型投資）

**条件（いずれか該当）:**
1. 投資額が100万円〜1,000万円
2. 新規技術スタックの採用
3. 複数ドメインに影響を与える投資
4. 外部ベンダーとの1年以上の契約
5. 技術的リスクが「中」以上

**必要なアクション:**
- ビジネスケース（簡易版、1-2ページ）提出
  - 投資目的・期待効果
  - 代替案比較（最低2案）
  - 簡易ROI計算
  - リスク評価
- CTO承認（財務面はCFO確認）
- 新技術の場合: 技術評価委員会でのレビュー

**承認期限:** 申請から2週間以内に可否回答

**判断者:** CTO + CFO（技術評価委員会が必要な場合は+2週間）

**プロセス:**
```
[ビジネスケース作成] → [申請] → [技術評価（新技術の場合）] → [CTO/CFO承認] → [実行] → [月次進捗報告]
```

**例:**
- 新しいBI ツールの導入（500万円）
- 複数部門で使用する分析基盤構築（800万円）
- 新規クラウドサービスの採用（300万円/年）

#### 5.2 IT投資委員会承認必須（大型投資）

**条件（いずれか該当）:**
1. 投資額が1,000万円以上
2. 全社共通基盤への影響
3. 戦略的重要性が高い（Transform投資）
4. 複数年契約（総額500万円以上）
5. 組織変更を伴う投資

**必要なアクション:**
- フルビジネスケース提出（テンプレート準拠、5-10ページ）
  - エグゼクティブサマリー
  - 現状分析・課題定義
  - 提案ソリューション
  - 代替案比較（最低3案）
  - 詳細財務分析（ROI, NPV, IRR）
  - 実施計画・スケジュール
  - リスク評価と緩和策
  - 組織・プロセス影響分析
- IT投資委員会での承認プレゼンテーション（20分）
- 承認後の四半期毎進捗報告義務

**承認期限:** 次回委員会（月次開催、最大1ヶ月）

**判断者:** IT投資委員会（CTO, CFO, 事業部長）

**プロセス:**
```
[フルBCケース作成] → [委員会申請] → [事前レビュー] → [委員会承認] → [実行] → [四半期レビュー] → [ベネフィット測定]
```

**例:**
- 全社ERP刷新（3億円）
- データレイク構築（5,000万円）
- グローバル展開を伴うCRM導入（1億円）

### 6. 禁止・エスカレーション（レッドゾーン）

#### 6.1 絶対禁止事項

以下は承認プロセス違反として厳格に禁止:

| 禁止事項 | 理由 | 違反時の措置 |
|---------|------|-------------|
| ビジネスケースなしの大型投資実施 | ガバナンス崩壊 | 投資即時停止、ドメインリーダーへ警告 |
| 承認前の契約締結 | 法的リスク | 契約無効化、責任者への懲戒 |
| 予算未確保での投資着手 | 財務規律違反 | プロジェクト中止、次年度予算減額 |
| 分割発注による承認回避 | 意図的ガバナンス迂回 | 関連投資の合算評価、厳重注意 |
| 投資台帳への未登録 | 可視性欠如 | 是正指導、再発時は予算執行停止 |

#### 6.2 技術的強制（自動チェック）

以下は技術的に自動チェックされ、違反は実行不可:
- 未承認クラウドアカウントでのリソース作成
- 予算上限超過の支出
- 未登録ベンダーとの契約手続き

#### 6.3 即時エスカレーション事項

以下は発見次第、即座にCTOへエスカレーション:
- 予算超過が見込まれるプロジェクト（10%以上）
- スケジュール遅延（1ヶ月以上）
- 重大なリスクの顕在化
- ベンダー契約トラブル

## ガードレール運用

### 7. 例外プロセス

#### 7.1 緊急投資（セキュリティ、障害対応等）

**適用条件:**
- 事業継続に即座の影響（システム停止、重大セキュリティリスク等）
- 24時間以内の対応が必要

**プロセス:**
1. CTO/CFOへ口頭または電話で状況報告
2. 口頭承認で着手可能
3. **48時間以内**に事後承認申請書を提出
4. 次回IT投資委員会で正式承認

**上限:** 500万円（超過する場合は経営層承認）

**記録:** すべての緊急投資は四半期レビューで報告

#### 7.2 ガードレール例外申請

**申請条件:**
- ガードレール逸脱の正当な理由（ビジネス機会損失、技術的制約等）
- リスク評価と緩和策の明示
- 代替案検討の証跡

**承認者:**
- 小型投資のガードレール逸脱 → CTO
- 中型投資のガードレール逸脱 → IT投資委員会
- 大型投資のガードレール逸脱 → IT投資委員会 + 経営会議

**申請書類:**
- 例外申請書（テンプレート使用）
- リスク評価シート
- 緩和策実施計画

**期限:** 通常承認プロセス+1週間

### 8. 投資優先順位付けプロセス

#### 8.1 ポートフォリオレビュー

**頻度:** 四半期毎
**参加者:** IT投資委員会メンバー
**実施事項:**

1. **ポートフォリオバランス確認**
   - Run/Grow/Transform 比率
   - 目標との乖離分析
   - 必要に応じた再配分

2. **進行中プロジェクトの健全性チェック**
   - RAGステータス（Red/Amber/Green）
   - 予算執行率
   - スケジュール遵守率
   - ベネフィット実現見込み

3. **新規案件の優先順位再評価**
   - 待機案件のレビュー
   - 市場環境変化の反映
   - リソース制約の考慮

#### 8.2 優先順位付けマトリクス

| ビジネス価値 ＼ 実現可能性 | 低 | 高 |
|------------------------|----|----|
| **高** | Transform優先<br>（戦略的投資） | 最優先実施<br>（Quick Win） |
| **低** | 後回し<br>（要再検討） | Grow優先<br>（効率化） |


#### 8.3 リソース配分原則

1. **Run投資**: 必要最小限を優先確保（品質・セキュリティ維持）
2. **Grow投資**: ROIが高い順
3. **Transform投資**: 戦略適合度が高い順
4. **リソース制約時**: 
   - 小さく始められる案件を優先
   - 段階的投資が可能な案件を優先
   - 外部リソース活用を検討

### 9. モニタリングと報告

#### 9.1 投資台帳

**目的:** 全投資の一元管理と可視性確保

**記録項目:**
- 投資ID、名称、カテゴリ（Run/Grow/Transform）
- 投資額、承認日、承認者
- 担当ドメイン、プロジェクトマネージャー
- ステータス（計画/実行中/完了/中止）
- 予算執行額、残額
- 期待ベネフィット、実現ベネフィット

**更新頻度:** 月次（最低）

**管理責任:** 中央PMO

#### 9.2 報告義務

| 投資分類 | 報告頻度 | 報告内容 | 報告先 |
|---------|---------|---------|-------|
| 小型 | 四半期 | サマリーレポート（件数、総額、カテゴリ別内訳） | IT企画部門 |
| 中型 | 月次 | 簡易進捗レポート（予算、スケジュール、リスク） | CTO |
| 大型 | 月次 | 詳細進捗レポート + 四半期委員会報告 | IT投資委員会 |
| 戦略的 | 月次 | 詳細進捗レポート + 経営会議報告 | 経営会議 |

**レポートテンプレート:** 中央PMOが提供

#### 9.3 コンプライアンス測定

**KPI:**
- **ガードレール遵守率**: 目標95%以上
  - 測定方法: (遵守案件数 / 全案件数) × 100%
  - 測定頻度: 四半期
  
- **承認プロセス違反件数**: 年間0件目標
  - 測定方法: 監査・内部通報による発覚件数
  - 測定頻度: 月次カウント、四半期レビュー

- **投資台帳登録率**: 100%
  - 測定方法: 支出記録との突合
  - 測定頻度: 月次

- **承認期限遵守率**: 90%以上
  - 測定方法: (期限内承認数 / 全申請数) × 100%
  - 測定頻度: 四半期

**是正措置:**
- 遵守率90%未満: 原因分析と改善計画策定
- 遵守率80%未満: ガードレール見直し検討

#### 9.4 ダッシュボード

中央PMOは以下の情報をリアルタイムダッシュボードで提供:
- ポートフォリオバランス（Run/Grow/Transform）
- 投資規模別内訳
- ドメイン別投資額
- 予算執行率
- プロジェクト健全性（RAGステータス）
- ガードレール遵守率

**アクセス権限:**
- IT投資委員会メンバー: 全データ
- ドメインリーダー: 自ドメインデータ + 全社サマリー
- プロジェクトマネージャー: 担当プロジェクトデータ
