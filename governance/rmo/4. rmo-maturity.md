# 記録管理マチュリティモデル：概要と評価の進め方

## 1｜マチュリティモデルとは

記録管理の改善を進めるうえで最初に直面する問題は、「何から手をつけるべきか」が見えないことです。システム・規程・プロセス・人材育成など、課題は複数の領域にまたがっており、さらに日本・北米・EMEAの3リージョンでは規制環境や運用実態も異なります。

マチュリティモデルは、こうした多面的・多地域にわたる現状を共通の物差しで可視化するための枠組みです。リージョンを横並びで評価することで、「グローバルで一斉に標準化すべき領域」と「リージョンごとに段階的に改善すべき領域」を意図的に区別できるようになります。また、一度きりの評価ではなく、PDCAサイクルに沿って繰り返し測定・改善していく継続的な活動の基盤として機能します。

本イニシアチブでは、ISO 30301をはじめとする国際標準規格を参照しつつ、自社のグローバルポリシーへの準拠と、これまでの記録管理への取り組みをレガシーとして継承した上で、自社独自の標準マチュリティモデルを定義します。このモデル自体が本イニシアチブの重要な成果物の一つであり、3リージョンにわたる継続的改善活動の共通基盤となります。

<br>

## マチュリティモデル（草案）

| レベル | 名称 | 状態の概要 |
|--------|------|-----------|
| Lv1 | 管理されていない | アドホック管理。文書が散在・重複し、検索・追跡が困難 |
| Lv2 | 初期段階 | 基本ルールと保管方法（フォルダ構造・命名規則）が整備されている |
| Lv3 | 定義済み | プロセスが標準化され、グローバルを含む組織横断で適用されている |
| Lv4 | 管理されている | KPIに基づき管理。DMS/ECMシステムが統合運用されている |
| Lv5 | 最適化されている | 継続的改善・自動化・AIを活用したプロアクティブな管理 |

> 当面の目標はLv3到達（プロセスの標準化とグローバル適用）。Lv4以降は中長期フェーズで検討。

<br>

### TRANSLATE

| Level | Name | Description |
|-------|------|-------------|
| Lv1 | Unmanaged | Ad hoc management. Records are scattered and duplicated, making retrieval and tracking difficult. |
| Lv2 | Initial | Basic rules and storage methods (folder structures, naming conventions) are established. |
| Lv3 | Defined | Processes are standardized and applied organization-wide, including globally. |
| Lv4 | Managed | KPI-based management with integrated DMS/ECM system operations. |
| Lv5 | Optimizing | Proactive management leveraging continuous improvement, automation, and AI. |

## 2｜5つの評価観点

マチュリティの評価は、以下5つの観点それぞれについて実施します。単一のスコアではなく、観点ごとに現在地を把握することで、どこに優先課題があるかを明確にします。

| 観点 | 評価の焦点 |
|------|-----------|
| ① ポリシー・規程 | 文書ライフサイクルの定義、保存期間・廃棄基準、RACIマトリクスの整備状況 |
| ② システム | SharePoint Online等DMS/ECMの導入・活用状況、検索性、アクセス制御 |
| ③ プロセス | 版管理・承認フロー・アーカイブ／廃棄ルールの実運用状況 |
| ④ 人・文化 | 教育・研修の実施状況、現場でのコンプライアンス意識と実効性 |
| ⑤ モニタリングと改善 | KPIの設定・測定状況、監査サイクル、是正・予防措置の運用 |

## 3｜観点×レベル マトリクス（現在地議論の素材）

各観点でLv1〜Lv5の典型的な状態を示します。自社がどこに当てはまるかを議論する素材として活用してください。

### ① ポリシー・規程

| Lv | 典型的な状態 |
|----|------------|
| 1 | 規程なし、または10年以上更新されていない旧規程のみ存在 |
| 2 | 保存期間・廃棄ルールに関する規程がリージョンまたは一部拠点のみで整備されている |
| 3 | グローバル共通ポリシーに準拠しつつ、各国・地域の法規制に対応した個別管理規定が策定されており、<br>RACIによって役割責任が明確になっている |
| 4 | 規程の定期レビュー・改訂プロセスが確立。年次でステークホルダー承認 |
| 5 | 規制変化・リスク動向に応じた動的更新。自動通知・バージョン管理完備 |

### ② システム

| Lv | 典型的な状態 |
|----|------------|
| 1 | 共有フォルダ・メール中心。SharePointが導入されていても活用不十分 |
| 2 | SharePointの基本運用（ライブラリ・メタデータ）が一部拠点で稼働 |
| 3 | ステータス管理・承認ワークフローが導入済み。多言語・クロスボーダーアクセス対応 |
| 4 | ERPや電子署名との連携。グローバルシングルサインオン・アクセス制御が実用的 |
| 5 | AI検索・自動分類・予測アーカイブ。自然言語検索によるクロスシステム運用 |

### ③ プロセス

| Lv | 典型的な状態 |
|----|------------|
| 1 | 版管理は手動・任意。廃棄基準なく複数バージョンが混在 |
| 2 | 基本的な承認フローと廃棄ルールが存在。紙・電子が混在 |
| 3 | 文書・記録のライフサイクルが完全実装。捕捉→承認→保管→廃棄が一貫して運用 |
| 4 | 自動ワークフロー・期限アラートで承認遅延がゼロに近い状態 |
| 5 | 機械学習による予測廃棄・自動アーカイブが稼働 |

### ④ 人・文化

| Lv | 典型的な状態 |
|----|------------|
| 1 | 意識が低く現場任せ。ルール違反が多発 |
| 2 | 新人研修のみ実施。国内中心で海外拠点は未対応 |
| 3 | 全社eラーニング・グローバル研修が整備。海外拠点を含む |
| 4 | 年次意識調査と改善施策が定着。遵守率90%以上 |
| 5 | 文化として定着。現場からのボトムアップ改善提案が活発 |

### ⑤ モニタリングと改善

| Lv | 典型的な状態 |
|----|------------|
| 1 | 監査なし。問題の把握が不能な状態 |
| 2 | 年次内部監査を実施。指摘事項が多い |
| 3 | 月次KPI（遵守率・検索時間等）をダッシュボードで可視化。PDCAが初回転 |
| 4 | リアルタイム監視・自動アラートが稼働。監査指摘がほぼゼロ |
| 5 | 予測分析とAI異常検知によるプロアクティブな改善が実現 |

## 4｜評価方法

### スコアリングの考え方

各観点（5つ）× 各リージョン（日本・北米・EMEA）の組み合わせで、それぞれLv1〜5のスコアを付与します。スコアは「現状の証拠に基づく合意形成」を原則とし、担当者の主観ではなく、後述の評価手段から得られた事実を根拠とします。

| 評価軸 | 内容 |
|--------|------|
| スコア単位 | 各観点×リージョンごとに1〜5点（整数）で評価 |
| 判定基準 | 各レベルの「典型的な状態」（セクション3のマトリクス）に照らして判定 |
| 集計 | 観点別・リージョン別・全体の3層で可視化 |

### 評価手段

単一の手段ではなく、以下を組み合わせることで、評価の客観性と網羅性を担保します。

| 手段 | 目的 | 主な対象 |
|------|------|---------|
| インタビュー | 現場の実態・認識を把握 | 各リージョンのRMO担当・業務部門 |
| ドキュメントレビュー | 規程・手順書・システム設定等の実在と内容を確認 | 文書管理規程、SharePoint構成、監査記録等 |
| 自己評価アンケート | リージョン間の比較に向けた定量的な一次情報収集 | 各リージョン担当者 |
| ログ分析 | 文書記録・システムログ等のデータを分析し、モニタリング観点の実態を客観的に評価 | 遵守率・監査指摘件数・廃棄実施率等、SharePointアクセスログ |

---

### リージョン間の比較方法

3リージョンは同一のスコアリング基準・評価手段を用いることで、横並び比較が可能な共通の物差しを担保します。比較結果は以下の2軸で分析します。

- リージョン間のばらつき：同一観点でスコアが乖離している領域 → リージョン固有の改善課題として整理
- リージョン共通の低スコア領域：全リージョンで低い観点 → グローバル一斉改善の優先候補として整理

## 5｜成果物の定義

本イニシアチブの評価フェーズで作成する成果物を以下に定義します。これらは単発の報告書ではなく、PDCAサイクルを通じて更新・活用され続けるグローバル改善活動の基盤として位置づけます。

| 成果物 | 内容 | 活用目的 |
|--------|------|---------|
| ① 標準マチュリティモデル定義書 | 自社独自の評価基準（5観点×5レベルの定義、スコアリングルール）を文書化したもの。ISO 30301・グローバルポリシー・既存の取り組みを統合して策定 | 3リージョン共通の評価基盤。サイクルごとに参照・改訂 |
| ② リージョン別スコアリング結果（ダッシュボード） | 日本・北米・EMEAの各観点スコアをレーダーチャートおよび一覧表で可視化したもの | 現在地の共有、リージョン間比較、改善進捗のトラッキング |
| ③ ギャップ分析レポート | 現状スコアと目標レベル（Lv3）のギャップを観点別・リージョン別に整理したもの | 改善優先度の根拠資料 |
| ④ グローバル／リージョン別改善アクションプラン | ギャップ分析に基づき、グローバルで一斉に取り組む施策とリージョン固有の施策を区別して整理したロードマップ | 改善活動の実行管理・進捗レビューの基準 |
